## Overview

Moneta's web platform enables state governments in Nigeria to manage tax collection and revenue governance processes. When I was brought in, the existing interface was creating measurable operational bottlenecks. Government administrators were working around the system rather than through it, and security vulnerabilities in the platform were creating exposure on high-value revenue flows.

## Constraints

The redesign had to be completed in two weeks. That ruled out foundational user research and custom component development. I used a pre-existing UI kit and design system to establish visual consistency from day one, which freed the available time for the work that actually mattered: understanding where the bottlenecks were and designing solutions that could be built within the existing API surface.

## Process

I worked directly with the engineering team to map the existing bottlenecks in the application. Several of the problems users were experiencing were not purely interface problems. They reflected gaps between what the UI implied was possible and what the underlying APIs actually supported. For each new feature added in the redesign, I analyzed the available API responses before designing the interface, ensuring that what I designed could be built without requiring backend changes that would blow the deadline.

I also provided ongoing support to the frontend development team during implementation, supplying resources and design specifications that reduced the back-and-forth that typically slows handoff.

## Outcomes

Contributed to over 50 billion naira in government revenue savings through improved workflow efficiency and remediation of security vulnerabilities

Measurable increase in user activity following launch, driven by elimination of existing workflow bottlenecks

Delivered within the two-week deadline
